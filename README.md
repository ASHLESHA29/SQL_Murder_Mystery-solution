# SQL Murder Mystery - Case Study

## Objective
Solve a fictional murder case using SQL queries and data analysis.

---

## Dataset Overview
The database contains multiple tables including:

- person
- crime_scene_report
- interview
- drivers_license
- get_fit_now_member
- get_fit_now_check_in
- facebook_event_checkin

These tables were analyzed to trace the criminal.

---

## Investigation Process

### 1. Crime Scene Analysis
Identified murder details from `crime_scene_report`.

### 2. Witness Identification
Found two witnesses based on address information.

### 3. Interview Analysis
Extracted clues related to gym membership and vehicle plate.

### 4. Suspect Identification
Matched gym members with vehicle data.

### 5. First Suspect
Jeremy Bowers was identified as the direct killer.

### 6. Mastermind Investigation
Jeremy revealed he was hired by a wealthy woman.

### 7. Final Suspect
Used physical description and event attendance data to find the mastermind.

---

## Final Result

| Role | Name |
|------|------|
| Killer | Jeremy Bowers |
| Mastermind | Miranda Priestly |
